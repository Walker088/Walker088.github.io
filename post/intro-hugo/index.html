使用 Hugo 打造個人部落格

@Walker088 · Jan 5, 2019 · 4 min read

1. 靜態網頁產生器

目前最流行的3個靜態網頁產生器分別是：

  • Hexo

    Hexo 是使用nodejs開發的靜態網頁框架，因為對nodejs語法比較陌生因此比較沒有考慮。 比較特別的是開發者是台灣人！！要是用Golang寫的可能就入坑啦。

  • Jekyll

    Jekyll 是使用ruby撰寫的靜態網頁產生器，更是Github推薦的御用部落格框架， 因此在這三款平台中Github星星數也是最多的。早前我也是使用Jekyll來寫部落格， 但是因為Jekyll要換Template相當麻煩，因此就跳槽到了本篇的主題 – Hugo 平台。

  • Hugo

    Hugo 是使用Golang開發的網誌框架，Github上的星星數比Hexo略多， 個人滿喜歡的特色之一是，Template的轉換只需要再設定檔裡面更改名字即可， 不像Jekyll手續複雜。此外因為是用Golang撰寫，在編譯速度上也是樂勝樓上兩個工具， 因此今天的分享就來看看怎麼使用Hugo迅速起手撰寫部落格啦。

2. 安裝 Hugo

今天使用的系統是MacOS，因此接下來的分享用到的指令會是Mac版本的， 如果你使用的是其他的作業系統，則可以參考 官方文件的教學。 我們選擇使用macOS下知名的套件管理工具Homebrew來安裝Hugo， 如果你的電腦裡面已經有安裝過Homebrew了， 就可以直些從下面 brew install 的部分開始操作。

# If you haven't installed homebrew
$ mkdir homebrew && curl -L https://github.com/Homebrew/brew/tarball/master | tar xz --strip 1 -C homebrew
# Install hugo via homebrew
$ brew install hugo
# Check whether hugo is be installed
$ hugo version

3. 建立及設定部落格專案

我們先使用 hugo 命令新增一個空白專案，然後下載一個Template到我們的專案裡面， 接著新增四個我們想加到模板 Menu 的頁面: about, history, tags, categories. 最後則是新增一篇空白的文章到專案內。

# 1. create the project
$ hugo new site myblog
# 2. add a theme
$ git submodule add https://github.com/laozhu/hugo-nuo themes/hugo-nuo
# 3. add new pages
$ hugo new about.md
$ hugo new hisroty.md
$ hugo new tags.md
$ hugo new categories.md
# 4. add new article
$ hugo new post/welcome.md

接下來我們需要修改專案的設定檔 config.toml， 裡面的內容相當清楚，包括給部落格取名、指定Template名稱等等。

# ./config.toml
baseURL = "https://{{your_site}}"
languageCode = "en-us"
title = "{{Your blog's name}}"
theme = "hugo-nuo"
PygmentsCodeFences = true # make code block more colorful
enableRobotsTXT = true

除了上面的基本設定外，還有一些額外的設定可以使用。 注意用中括號包起來的部分是你需要填入自己資料的地方。 關於這份模板可以使用的設定可以 參考它文件的說明， 或者在 ./themes/hugo-nuo/exampleSite/config.toml 這份文件裡面也有其他參數的範例。

# ./config.toml
baseURL = "https://{{your_site}}"
languageCode = "en-us"
title = "{{your blog's name}}"
theme = "hugo-nuo"
PygmentsCodeFences = true
googleAnalytics = "{{Google Analytics UA number}}"
disqusShortname = "{{Disqus Username}}"
enableRobotsTXT = true

[menu]
  [[menu.main]]
    identifier = "home"
	name = "Home"
	url = "/"
	weight = 1 # weight用來調整Link在側邊菜單的順序
  [[menu.main]]
    identifier = "categories"
	name = "Categories"
	url = "/categories/"
	weight = 2
  [[menu.main]]
    identifier = "history"
	name = "History"
	url = "/history/"
	weight = 3
  [[menu.main]]
    identifier = "tags"
	name = "Tags"
	url = "/tags/"
	weight = 4
  [[menu.main]]
    identifier = "about"
	name = "About"
	url = "/about/"
	weight = 5

[params]
  seotitle = "Hugo Blog (SEO Version)"
  subtitle = "- work hard and play hard -"
  description = "Hugo is one of the most popular open-source static site generators. With its amazing speed and flexibility, Hugo makes building websites fun again."

  email = "{{your email}}"
  github = "{{your github id}}"
  linkedin = "{{your linkedin id}}"
  twitter = "{{your twitter id}}"
  instagram = "{{your ig id}}"

接下來我們可以替剛剛新增的文章 welcom.md 加上分類以及標籤方便查找：

# ./content/post/welcome.md
---
title: Welcome
author: "Walker088"
categories: "hugo"
tags: ["blog", "hugo"]
date: 2019-01-05
draft: true
---
Welcom to my blog !!!

完成上述工作後就可以先在本地端查看結果了! 在terminal內下完啟動伺服器的命令後，在瀏覽器輸入 http://localhost:1313 就可以看到我們剛剛的成果了。

$ hugo server -D # -D 參數表示顯示草稿

這邊要注意我們剛剛新增的頁面 about, history, tags, categories 和文章 welcome.md 在 header 上預設 *draft: true*， 在正式部署前需要改成 false 才可以正確顯示哦！

4. 新增 RSS 訂閱功能

在這份模板內也有提供 RSS 訂閱的功能 (但在他的教學文件內並沒有提到，因為並不完整的關係，需要自己做一些修改，應該算是隱藏版功能摁摁)。 首先在設定檔 ./config.toml 內加入 rss 的設定 (最後一行)：

# ./config.toml
[params]
  seotitle = "Hugo Blog (SEO Version)"
  subtitle = "- work hard and play hard -"
  description = "Hugo is one of the most popular open-source static site generators. With its amazing speed and flexibility, Hugo makes building websites fun again."

  email = "{{your email}}"
  github = "{{your github id}}"
  linkedin = "{{your linkedin id}}"
  twitter = "{{your twitter id}}"
  instagram = "{{your ig id}}"
  
  # e.g., http://{{user_name}}.github.io
  rss = "http://{{your_site}}/index.xml"

接著在新增一個產生 rss 的模板在專案目錄下： ./layout/index.rss.xml ，這部分我們可以 參考官網的RSS模板教學。 這邊有個小小的更動是在迴圈產生 <item> 標籤的部分，為了避免連同我們的頁面都被加入 rss 檔案內因此新增了一個過濾器。

<rss version="2.0" xmlns:atom="http://www.w3.org/2005/Atom">
  <channel>
    <title>{{ if eq  .Title  .Site.Title }}{{ .Site.Title }}{{ else }}{{ with .Title }}{{.}} on {{ end }}{{ .Site.Title }}{{ end }}</title>
    <link>{{ .Permalink }}</link>
    <description>Recent content {{ if ne  .Title  .Site.Title }}{{ with .Title }}in {{.}} {{ end }}{{ end }}on {{ .Site.Title }}</description>
    <generator>Hugo -- gohugo.io</generator>{{ with .Site.LanguageCode }}
    <language>{{.}}</language>{{end}}{{ with .Site.Author.email }}
    <managingEditor>{{.}}{{ with $.Site.Author.name }} ({{.}}){{end}}</managingEditor>{{end}}{{ with .Site.Author.email }}
    <webMaster>{{.}}{{ with $.Site.Author.name }} ({{.}}){{end}}</webMaster>{{end}}{{ with .Site.Copyright }}
    <copyright>{{.}}</copyright>{{end}}{{ if not .Date.IsZero }}
    <lastBuildDate>{{ .Date.Format "Mon, 02 Jan 2006 15:04:05 -0700" | safeHTML }}</lastBuildDate>{{ end }}
    {{ with .OutputFormats.Get "RSS" }}
        {{ printf "<atom:link href=%q rel=\"self\" type=%q />" .Permalink .MediaType | safeHTML }}
    {{ end }}
    {{ range first 15 (where .Data.Pages "Section" "post") }}
    <item>
      <title>{{ .Title }}</title>
      <link>{{ .Permalink }}</link>
      <pubDate>{{ .Date.Format "Mon, 02 Jan 2006 15:04:05 -0700" | safeHTML }}</pubDate>
      {{ with .Site.Author.email }}<author>{{.}}{{ with $.Site.Author.name }} ({{.}}){{end}}</author>{{end}}
      <guid>{{ .Permalink }}</guid>
      <description>{{ .Summary | html }}</description>
    </item>
    {{ end }}
  </channel>
</rss>

5. 部署到 Github Page

我們使用Github Page把網誌部署到網路上。 首先使用 hugo 指令將我們網站打包到專案目錄下的 ./public 資料夾內， 然後在Github上創建一個 Repo ，命名成 {{username}}.github.io (注意這邊中括號內代換成你 github 的帳號名稱)。 將這個新建的Repo clone 下來，把剛剛打包好的 public 資料夾內的內容通通複製過去， 再 push 回 github repo 後就完成了。

$ hugo # 將網站內容通通包到 ./public 資料夾下
$ git clone https://github.com/{{username}}/{{username}}.github.io.git ../
$ cp -r public ../{{username}}.github.io.git/
$ cd ../{{username}}.github.io.git/
$ git add . && git commit -m "new post"
$ git push -u origin

完成上傳的動作後，我們就可以在瀏覽器輸入 {{username}}.github.io 瀏覽我們剛剛上傳的網頁了！
